# 局域网聊天系统 (Client-Server)

基于 **TCP Socket** 的局域网即时通讯程序，支持多人聊天。

---

## 项目结构

```
ComputerNetworkServe/
├── server.py          # 服务器端程序
├── client.py          # 客户端程序
└── README.md          # 本文档
```

---

## 快速启动

### 前提条件

- Python 3.7+（无需安装第三方库，仅使用标准库）

### 步骤

1. **启动服务器**（在一台电脑上）

   ```bash
   python server.py
   ```

   或双击 `start_server.bat`
2. **启动客户端**（可以在多台电脑上启动）

   ```bash
   python client.py
   ```

   或双击 `start_client.bat`
3. 客户端输入服务器的 **局域网 IP** 和 **端口号**（默认 9999），输入用户名后即可聊天

### 客户端命令

| 命令        | 说明                 |
| ----------- | -------------------- |
| `/online` | 查看当前在线用户列表 |
| `/quit`   | 退出聊天室           |

## 基本原理

### 1. 网络通信模型：TCP Client-Server

```
  ┌──────────┐         ┌──────────────┐         ┌──────────┐
  │ 客户端 A  │◄───────►│   服务器      │◄───────►│ 客户端 B  │
  │ (Client)  │  TCP    │   (Server)   │  TCP    │ (Client)  │
  └──────────┘ 连接     └──────────────┘ 连接     └──────────┘
                              │
                              ▼
                        消息转发/广播
```

- **服务器 (Server)**：运行在一台电脑上，监听指定端口，等待客户端连接
- **客户端 (Client)**：运行在另一台电脑上，主动连接到服务器
- 所有消息通过服务器中转，客户端之间不直接通信

### 2. TCP 三次握手（建立连接）

```
客户端                        服务器
  │                            │
  │ ─── SYN ─────────────────► │  ① 客户端发起连接请求
  │                            │
  │ ◄── SYN + ACK ──────────── │  ② 服务器同意并确认
  │                            │
  │ ─── ACK ─────────────────► │  ③ 客户端确认，连接建立
  │                            │
  │ ◄════ 双向数据传输 ════════► │
```

### 3. Socket 编程流程

#### 服务器端

```
socket()    → 创建套接字
    ↓
bind()      → 绑定 IP 地址和端口
    ↓
listen()    → 开始监听连接请求
    ↓
accept()    → 接受客户端连接（阻塞等待）
    ↓
recv/send   → 接收/发送数据
    ↓
close()     → 关闭连接
```

#### 客户端

```
socket()    → 创建套接字
    ↓
connect()   → 连接到服务器
    ↓
send/recv   → 发送/接收数据
    ↓
close()     → 关闭连接
```

### 4. 多线程并发模型

服务器使用 **多线程 (threading)** 实现并发：

- **主线程**：循环调用 `accept()` 等待新客户端连接
- **子线程**：每个客户端连接分配一个独立线程 (`handle_client`)，负责接收该客户端的消息并广播给其他客户端

客户端也使用多线程：

- **主线程**：读取键盘输入并发送消息
- **接收线程**：持续监听服务器推送的消息并打印

### 5. 关键概念

| 概念              | 说明                                        |
| ----------------- | ------------------------------------------- |
| **Socket**  | 网络通信的端点，提供进程间跨网络通信的接口  |
| **TCP**     | 传输控制协议，提供可靠、有序的字节流传输    |
| **IP 地址** | 标识网络中的一台主机 (如 `192.168.1.100`) |
| **端口号**  | 标识主机上的一个进程/服务 (如 `9999`)     |
| **线程**    | 轻量级执行单元，实现并发处理多个客户端      |
| **广播**    | 服务器将消息转发给所有（或指定）客户端      |

## 代码功能详解

### server.py — 服务器

| 函数                | 功能                                                           |
| ------------------- | -------------------------------------------------------------- |
| `start_server()`  | 创建 Socket → 绑定地址 → 监听端口 → 循环接受连接            |
| `handle_client()` | 处理单个客户端：接收用户名 → 循环接收消息 → 广播 → 断开清理 |
| `broadcast()`     | 将消息发送给所有在线客户端（可排除发送者）                     |
| `remove_client()` | 安全关闭并移除一个客户端连接                                   |

### client.py — 客户端

| 函数                   | 功能                                                     |
| ---------------------- | -------------------------------------------------------- |
| `start_client()`     | 获取连接信息 → 连接服务器 → 发送用户名 → 启动收发循环 |
| `receive_messages()` | 接收线程函数，持续从服务器读取消息并打印                 |

## 注意事项

1. **防火墙**：确保服务器端的 9999 端口在 Windows 防火墙中放行

   ```
   # 以管理员身份运行 PowerShell
   New-NetFirewallRule -DisplayName "Chat Server" -Direction Inbound -Protocol TCP -LocalPort 9999 -Action Allow
   ```
2. **查看本机 IP**：在命令行运行 `ipconfig`，找到局域网适配器的 **IPv4 地址**
3. **同一台电脑测试**：可以开多个终端窗口，服务器 IP 填 `127.0.0.1`

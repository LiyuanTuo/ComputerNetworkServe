# 局域网聊天系统 (Client-Server)

基于 **TCP Socket** 的局域网即时通讯程序，支持多人聊天。

---

## 项目结构

```
ComputerNetworkServe/
├── server.py          # 服务器端程序
├── client.py          # 客户端程序
└── README.md          # 本文档
```

---

## 快速启动

### 前提条件

- Python 3.7+
- 实时语音功能需要安装 PyAudio

### 步骤

1. **启动服务器**（在一台电脑上）

   ```bash
   python server.py
   ```

   或双击 `start_server.bat`
2. **启动客户端**（可以在多台电脑上启动）

   ```bash
   python client.py
   ```

   或双击 `start_client.bat`
3. 客户端输入服务器的 **局域网 IP** 和 **端口号**（默认 11451），输入用户名后即可聊天

### 默认端口规划

| 用途 | 协议 | 默认端口 |
| ---- | ---- | -------- |
| 服务器控制连接 | TCP | `11451` |
| 会议室语音中继 | UDP | `8888` |
| 客户端本地会议语音 | UDP | `9999`（占用时顺延到固定候选池） |
| 客户端本地单聊语音 | UDP | `10000`（占用时顺延到固定候选池） |
| 服务器单聊语音中继 | UDP | `11452`-`11471` |

### 客户端命令

| 命令        | 说明                 |
| ----------- | -------------------- |
| `/online` | 查看当前在线用户列表 |
| `/quit`   | 退出聊天室           |
| `/voice`  | 语音广播        |
| `@P1` | 单独向P1发送文本 |
| `/voice @P1` | 单独向P1发送语音 |
| `/call @p1` | 单独向P1发起语音通话 |
| `/realtime -quit` | 挂断语音电话 |

## 基本原理

### 1. 网络通信模型：TCP Client-Server

```
  ┌──────────┐         ┌──────────────┐         ┌──────────┐
  │ 客户端 A  │◄───────►│   服务器      │◄───────►│ 客户端 B  │
  │ (Client)  │  TCP    │   (Server)   │  TCP    │ (Client)  │
  └──────────┘ 连接     └──────────────┘ 连接     └──────────┘
                              │
                              ▼
                        消息转发/广播
```

- **服务器 (Server)**：运行在一台电脑上，监听指定端口，等待客户端连接
- **客户端 (Client)**：运行在另一台电脑上，主动连接到服务器
- 所有消息通过服务器中转，客户端之间不直接通信

### 2. 实时语音通话流程（TCP 信令 + UDP 中转）

实时语音基于混合架构实现，TCP 用于控制信令，UDP 用于音频数据中继传输以保证低延迟。

```
客户端 A (主叫)                 服务器 (Server)                  客户端 B (被叫)
      │                               │                                │
      │ 1️⃣ 发起：/call @ClientB      │                                │
      ├────(TCP 信令)────────────────►│ 2️⃣ 校验并分配 UDP 中继端口       │
      │                               │                                │
      │                               │ 3️⃣ 邀请：通知 B 有语音呼叫        │
      │                               ├────(TCP 信令)─────────────────►│
      │                               │                                │
      │                               │ 4️⃣ 响应：同意接听 (/CALL_ACCEPT) │
      │                               │◄───(TCP 信令)──────────────────┤
      │                               │                                │
      │ 5️⃣ 确认：同意接听及 UDP 端口   │                                │
      │◄───(TCP 信令)─────────────────┤                                │
      │                               │                                │
      │               6️⃣ 建立 UDP 音频流通道并开始通话                   │
      │◄════(UDP 音频数据/中继转发)════►│◄════(UDP 音频数据/中继转发)═════►│
      │                               │                                │
      │ 7️⃣ 挂断：/realtime -quit     │                                │
      ├────(TCP 信令)────────────────►│                                │
      │                               │ 8️⃣ 终止：通知结束通话并清理端口   │
      │◄───(TCP 信令)─────────────────┤────(TCP 信令)─────────────────►│
```

具体步骤说明：
1. **呼叫发起**：主叫方（客户端 A）发送 `/call @用户名` 指令，通过 TCP 隧道将请求发送给服务器。
2. **分配中继通道**：服务器进行状态校验（是否在线、是否正忙）。若允许通话，则在服务器端固定 UDP 中继端口池中分配一个可用端口作为专属“中继服务”（Pending 状态）。
3. **呼叫邀请**：向被叫方（客户端 B）发送包含请求人与分配的中继 UDP 端口号的 `\CALL_REQUEST` TCP 信令。
4. **用户响应**：被叫方用户选择同意或拒绝。同意后通过 TCP 发回 `\CALL_ACCEPT` 指令。
5. **打通语音流**：服务器收到被叫方同意后，将其加入活跃通话队列并启动服务端的专门中继线程；同时通过 TCP 信令向客户端 A 下发对方已就绪及中继 UDP 端口。
6. **语音交互**：双方使用本地麦克风与扬声器采集/播放，将 PCM 音频封装打包，以高频 UDP 数据报的形式通过服务器的指定端口进行双向转发（Server Relay）。
7. **挂断清理**：任一方发送 `/realtime -quit`，服务器收到该 TCP 指令后，停止对应的 UDP 中继线程并关闭释放该端口，向双方下发通话结束通知，本地停止录音播音。

### 2.1 自适应语音传输

实时语音链路现在包含一条“接收端评估、发送端调档”的自适应闭环：

1. 接收端在最近几秒的滚动窗口内统计丢包率、时延、抖动和乱序率。
2. 接收端根据网络质量生成建议档位，并通过 UDP 控制报文反馈给发送端。
3. 发送端按最近收到的反馈自动切换编码档位；多人会议时取最近反馈中的最保守档位。
4. 服务器仍然只负责 UDP 透明中继，不参与编解码和自适应决策。

当前默认档位如下：

| 档位 | 发送采样率 | 编码方式 | 适用网络 |
| ---- | ---------- | -------- | -------- |
| `good` | 16000 Hz | `pcm16` | 网络良好，优先保真 |
| `fair` | 12000 Hz | `ulaw8` | 网络一般，折中带宽与质量 |
| `poor` | 8000 Hz | `adpcm4` | 网络较差，优先稳态传输 |

`audioop` 不是可单独安装的 PyPI 包；在 Python 3.13+ 中它已从标准库移除，因此 `pip install audioop` 会失败。
若运行环境不支持 `audioop`，`fair/poor` 档会自动回退为 `zlib_pcm16` 压缩传输，无需额外安装该模块。

### 2.2 UDP 语音/控制报文格式

实时语音与自适应控制共用同一套 UDP 报头：

```text
[PRIORITY 1B][MAGIC 4B][JSON_LEN 4B][JSON_UTF8 N][PAYLOAD M]
```

关键 JSON 字段如下：

| 字段 | 说明 |
| ---- | ---- |
| `v` | 协议版本 |
| `sid` | 发送者 ID (`crc32(username)`) |
| `seq` | 序列号 |
| `ts` | 发送时间戳 |
| `kind` | `audio` 或 `control` |
| `codec` | `pcm16` / `ulaw8` / `adpcm4` / `zlib_pcm16` |
| `sr` | 当前音频负载采样率 |
| `profile` | 当前发送档位 |

当 `kind=control` 且 `control=adapt` 时，报文体为空，JSON 中额外携带 `recommend`、`loss`、`delay_ms`、`jitter_ms`、`reorder`、`score` 等反馈字段，用于驱动发送端调档。

### 3. TCP 三次握手（建立连接）

```
客户端                        服务器
  │                            │
  │ ─── SYN ─────────────────► │  ① 客户端发起连接请求
  │                            │
  │ ◄── SYN + ACK ──────────── │  ② 服务器同意并确认
  │                            │
  │ ─── ACK ─────────────────► │  ③ 客户端确认，连接建立
  │                            │
  │ ◄════ 双向数据传输 ════════► │
```

### 4. Socket 编程流程

#### 服务器端

```
socket()    → 创建套接字
    ↓
bind()      → 绑定 IP 地址和端口
    ↓
listen()    → 开始监听连接请求
    ↓
accept()    → 接受客户端连接（阻塞等待）
    ↓
recv/send   → 接收/发送数据
    ↓
close()     → 关闭连接
```

#### 客户端

```
socket()    → 创建套接字
    ↓
connect()   → 连接到服务器
    ↓
send/recv   → 发送/接收数据
    ↓
close()     → 关闭连接
```

### 5. 多线程并发模型

服务器使用 **多线程 (threading)** 实现并发：

- **主线程**：循环调用 `accept()` 等待新客户端连接
- **子线程**：每个客户端连接分配一个独立线程 (`handle_client`)，负责接收该客户端的消息并广播给其他客户端

客户端也使用多线程：

- **主线程**：读取键盘输入并发送消息
- **接收线程**：持续监听服务器推送的消息并打印

### 6. 关键概念

| 概念              | 说明                                        |
| ----------------- | ------------------------------------------- |
| **Socket**  | 网络通信的端点，提供进程间跨网络通信的接口  |
| **TCP**     | 传输控制协议，提供可靠、有序的字节流传输    |
| **IP 地址** | 标识网络中的一台主机 (如 `192.168.1.100`) |
| **端口号**  | 标识主机上的一个进程/服务 (如 `11451`)     |
| **线程**    | 轻量级执行单元，实现并发处理多个客户端      |
| **广播**    | 服务器将消息转发给所有（或指定）客户端      |

## 代码功能详解

### server.py — 服务器

| 函数                   | 功能                                                                 |
| ---------------------- | -------------------------------------------------------------------- |
| `start_server()`       | 创建 Socket → 绑定地址 → 监听端口 → 循环接受连接                   |
| `handle_client()`      | 处理单个客户端：接收用户名 → 循环接收消息 → 对应处理 → 断开清理    |
| `broadcast()`          | 将消息广播给所有在线客户端（可排除发送者）                           |
| `privatecast()`        | 将消息单独发送（私聊）给指定的主机目标                               |
| `remove_client()`      | 安全关闭并移除一个客户端连接，同时结束可能正在进行的通话             |
| `handle_call_request()`| 处理一端发起的实时语音请求，做校验并为其分配专属临时 UDP 中继端口    |
| `handle_call_reply()`  | 接收被叫端对语音请求的答复，同意后负责真正打通双方的中继映射通道     |
| `udp_voice_relay()`    | UDP 专用语音中继线程，高性能、低延迟地在对端间互相转发采集的音频数据报|
| `end_realtime_voice()` | 主动或被动终止指定用户的实时语音连接，清理系统内存映射并关闭对应的端口|

### client.py — 客户端

| 函数                   | 功能                                                           |
| ---------------------- | -------------------------------------------------------------- |
| `start_client()`       | 获取并验证连接信息 → 连接服务器 → 发送用户名 → 启动发信与指令解析循环 |
| `receive_messages()`   | 后台接收线程，持续从服务器读取 TCP 消息，并做富文本或信令动作解析打印 |

### audio.py — 音频处理模块

| 函数                        | 功能                                                                   |
| --------------------------- | ---------------------------------------------------------------------- |
| `record_audio()`            | 录制麦克风短音频并保存为本地的临时 `temp_voice.wav`，用于语音留言广播  |
| `play_audio()`              | 播放指定的本地 WAV 声音文件到扬声器                                    |
| `start_realtime_audio()`    | 开启局域网/公网自动 NAT 打洞缓冲，随后启动实时语音的双向收发子线程     |
| `stop_realtime_audio()`     | 终止实时语音收发线程，安全退出无限循环，释放音频流硬件和网络底层端口   |
| `udp_audio_send_thread()`   | 采音发包后台线程：从麦克风连续拾音(1024 chunks)并打包通过 UDP 直发服务器中继 |
| `udp_audio_recv_thread()`   | 收包播音后台线程：从网络层循环阻塞侦听到达的音频数据包并即刻播放至声卡，同时根据网络质量回传自适应控制帧 |
| `get_pyaudio()`             | 锁保护机制下的单例音频驱动访问实例获取，防止 C 层调用引发多线程崩溃    |

## 注意事项

1. **防火墙**：确保服务器端的 11451/TCP 端口以及会议室语音 8888/UDP 端口在 Windows 防火墙中放行

   ```
   # 以管理员身份运行 PowerShell
    New-NetFirewallRule -DisplayName "LinkVoice TCP Server" -Direction Inbound -Protocol TCP -LocalPort 11451 -Action Allow
    New-NetFirewallRule -DisplayName "LinkVoice Room UDP" -Direction Inbound -Protocol UDP -LocalPort 8888 -Action Allow
   ```
2. **查看本机 IP**：在命令行运行 `ipconfig`，找到局域网适配器的 **IPv4 地址**
3. **同一台电脑测试**：可以开多个终端窗口，服务器 IP 填 `127.0.0.1`
